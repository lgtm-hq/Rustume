Connected Rustume can publish a resume as a shareable read-only page. Publishing remains opt-in:
private account-backed resumes are not made public simply because they are synchronized.

## Publishing controls

- Publish or unpublish a selected resume with a unique public URL
- Protect a page with a password when the resume should be shared selectively
- Render previews and social metadata for an intentionally public page
- Configure a custom domain in deployments that provide one

Public pages, password protection, and custom-domain support are not paid-only functionality.
Rustume Cloud provides an operated deployment; self-hosted connected installations may expose the
same controls.

Before publishing, review the resume for phone numbers, email addresses, physical locations, and
other information intended only for a specific recipient. See [Encryption](/docs/cloud/encryption/)
for private stored content and [Cloud Storage](/docs/cloud/storage/) for account-backed data.
